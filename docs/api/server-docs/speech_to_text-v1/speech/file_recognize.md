# Audio file speech recognition (ASR)

An audio speech recognition API is provided to recognize the entire audio file (less than 60s) at one time.

> Single-tenant traffic limit: 20 QPS. Apps under the same tenant share the 20 QPS. The free version does not support calling this interface.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/speech_to_text/v1/speech/file_recognize |
| HTTP Method | POST |
| Supported app types | custom,isv |
| Required scopes | `speech_to_text:speech` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" Learn more about obtaining and using access_token. |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `speech` | `speech` | Yes | Audio resources |
|   `speech` | `string` | No | Content of the base64-encoded audio file **Example value**: "PdmrfE267Cd/Z9KpmNFh71A2PSJZxSp7+8upCg==" |
| `config` | `file_config` | Yes | Configures properties |
|   `file_id` | `string` | Yes | A 16-bit string generated by a user to identify a file. The string can only contain letters, numbers, and underscores. **Example value**: "qwe12dd34567890w" |
|   `format` | `string` | Yes | Audio format. Only pcm supported. **Example value**: "pcm" |
|   `engine_type` | `string` | Yes | Engine type. Only 16k_auto that allows a mix of Chinese and English is supported. **Example value**: "16k_auto" | ### Request body example

{
    "speech": {
        "speech": "PdmrfE267Cd/Z9KpmNFh71A2PSJZxSp7+8upCg=="
    },
    "config": {
        "file_id": "qwe12dd34567890w",
        "format": "pcm",
        "engine_type": "16k_auto"
    }
}

## Response

### Response body
| Parameter | Type | Description |
| --- | --- | --- |
| `code` | `int` | Error codes, fail if not zero |
| `msg` | `string` | Error descriptions |
| `data` | `\-` | \- |
|   `recognition_text` | `string` | Text after speech recognition | ### Response body example

{
    "code": 0,
    "msg": "success",
    "data": {
        "recognition_text": "Hello. You may want to try Lark."
    }
}

### Error code
| HTTP status code | Error code | Description | Troubleshooting suggestions |
| --- | --- | --- | --- |
| 400 | 1040101 | invalid param | Parameter error. Check the input parameters according to the appropriate document. |
| 500 | 1040102 | network anomaly | Service or network error. Request again. |
