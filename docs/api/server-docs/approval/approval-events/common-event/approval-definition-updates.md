# Approval definition updates
> To understand the usage scenes and configuration process of event subscription, click Event subscription overview.

This event is triggered when the "approval" definition is updated.

* Dependent permissions: Access Approval

**Callback example:**
```json
{
    "schema": "2.0",
    "header": {
        ## Unique event ID
        "event_id": "",
        "token": "",
        "event_type":"approval.approval.updated_v4" // eventType value
        "app_id": "cli_xxx",
        "tenant_key": "xxx"
        "event_create_time": "1502199207.7171419"，
    }
    "event": {
        "object": { // Current data, which is the standard event format
            "approval_id": "",    // Approval definition ID
            "approval_code": "",  // Approval definition code
            "version_id": "",  // Approval definition version number
            "widget_group_type": 0,  // The type of widgets; if the value is 0, it indicates that the widgets are unused
            "FormDefinitionId": "",  // Form definition ID
            "process_obj": "",  // Approval process design
            "timestamp": "", // Time
            "extra": ""  // Extension field
        }
     } 
}
```
