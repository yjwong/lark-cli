# Send a message

Sends messages to a specific user or chat, with text, rich text, interactive message cards, group cards, name cards, images, videos, audios, files, and emojis supported.

> Notes:
> - Bot ability must be enabled.
> - To send messages to a user, the bot must be available for the user.
> - To send messages to a group, the bot must be in the group chat.
> - To avoid disturbing users, the frequency limit for sending messages to the same user is 5 QPS, and the frequency limit for sending messages to the same group is 5 QPS shared by bots in the group.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/im/v1/messages |
| HTTP Method | POST |
| Rate Limit | 1000 per minute、50 per second |
| Supported app types | custom,isv |
| Required scopes Enable any scope from the list | `im:message` `im:message:send_as_bot` `im:message:send` |
| Required field scopes | > The response body of the API contains the following sensitive fields, and they will be returned only after corresponding scopes are added. If you do not need the fields, it is not recommended that you request the scopes. `contact:user.employee_id:readonly` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" How to choose and get access token |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Query parameters
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `receive_id_type` | `string` | Yes | ID type of the message receiver, which can be open_id, user_id, union_id, email, or chat_id **Example value**: open_id **Optional values are**:  - `open_id`: Identifies a user to an app. The same user has different Open IDs in different apps. How to get Open ID - `union_id`: Identifies a user to a tenant that acts as a developer. A user has the same Union ID in apps developed by the same developer, and has different Union IDs in apps developed by different developers. A developer can use Union ID to link the same user's identities in multiple apps. How to get Union ID - `user_id`: Identifies a user to a tenant. The same user has different User IDs in different tenants. In one single tenant, a user has the same User ID in all apps （including store apps）. User ID is usually used to communicate user data between different apps. How to get User ID. - `email`: Identifies users by "email", which is the user's actual email address - `chat_id`: Identifies group chats by chat_id. You can call the API Search for groups visible to a user or bot to obtain the chat_id value. For group ID description, refer to Group ID description.  **When the value is `user_id`, the following field scopes are required**: `contact:user.employee_id:readonly` | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `receive_id` | `string` | Yes | Fill in the corresponding message receiver ID based on the receive_id_type value. Open ID is recommended here. For details, refer to How to get Open ID? **Example value**: "ou_7d8a6e6df7621556ce0d21922b676706ccs" |
| `msg_type` | `string` | Yes | Message types include: text, post, image, file, audio, media, sticker, interactive, share_chat, and share_user; for the definitions of types, refer to Sent message content **Example value**: "text" |
| `content` | `string` | Yes | Message content, which is a string after the serialization of the JSON structure; different msg_type values correspond to different contents; message types include text, post, image, file, audio, media, sticker, interactive, share_chat, and share_user; for the description of the format, refer to Sent message content Notices: - The JSON string needs to be escaped, such as `\\n` for line breaks. - The maximum size of the text-message request body is 150 KB. - The maximum size of the card- and rich-text-message request body is 30 KB. **Example value**: "`{\"text\":\"test content\"}`" |
| `uuid` | `string` | No | A unique string sequence generated by the developer for request deduplication; Requests with the same uuid can successfully send at most one message within 1 hour. **Example value**: "a0d69e20-1dd1-458b-k525-dfeca4015204" **Data validation rules**: - Maximum length: `50` characters | ### Request body example

{
    "receive_id": "ou_7d8a6e6df7621556ce0d21922b676706ccs",
    "msg_type": "text",
    "content": "{\"text\":\"test content\"}",
    "uuid": "a0d69e20-1dd1-458b-k525-dfeca4015204"
}

**cURL Example**

```json 
curl --location --request POST 'https://open.larksuite.com/open-apis/im/v1/messages?receive_id_type=chat_id' \
--header 'Authorization: Bearer t-XXX' \
--header 'Content-Type: application/json; charset=utf-8' \
--data-raw '{
    "receive_id": "oc_84983ff6516d731e5b5f68d4ea2e1da5",
    "msg_type": "text",
    "content": "{\"text\":\"test content\"}"
}'
```

**Python Example**
```
import json
import requests

def send():
    url = "https://open.larksuite.com/open-apis/im/v1/messages"
    params = {"receive_id_type":"chat_id"}
    msg = "text content"
    msgContent = {
        "text": msg,
    }
    req = {
        "receive_id": "oc_xxx", # chat id
        "msg_type": "text",
        "content": json.dumps(msgContent)
    }
    payload = json.dumps(req)
    headers = {
        'Authorization': 'Bearer xxx', # your access token
        'Content-Type': 'application/json'
    }
    response = requests.request("POST", url, params=params, headers=headers, data=payload)
    print(response.headers['X-Tt-Logid']) # for debug or oncall
    print(response.content) # Print Response

if __name__ == '__main__':
    send()
```
