# Forward a message

Forwards a specified message to a user, group chat or thread.

> Notes:
> - Bot ability must be enabled.
> - To forward a message to a user, the bot must be available for the user.
> - To forward a message to the group, the bot needs to be in the group.
> - This API has the following restrictions:
> - Message types such as red envelopes, voting, audio and schedule transfers are not supported.
> - Re-forwarding a sub-message (the message with  upper_message_id field) in merge-forward messages is not supported.
> - To avoid disturbing users, the frequency limit for sending messages to the same user is 5 QPS, and the frequency limit for sending messages to the same group is 5 QPS shared by bots in the group.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/im/v1/messages/:message_id/forward |
| HTTP Method | POST |
| Rate Limit | 1000 per minute、50 per second |
| Supported app types | custom,isv |
| Required scopes Enable any scope from the list | `im:message` `im:message:send_as_bot` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" How to choose and get access token |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Path parameters
| Parameter | Type | Description |
| --- | --- | --- |
| `message_id` | `string` | ID of message to forward.  For the description, refer to Message ID description. **Example value**: "om_dc13264520392913993dd051dba21dcf" | ### Query parameters
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `receive_id_type` | `string` | Yes | ID type of the message receiver, which can be open_id, user_id, union_id, email, or chat_id **Example value**: open_id **Optional values are**:  - `open_id`: Identifies a user to an app. The same user has different Open IDs in different apps. How to get Open ID - `user_id`: Identifies a user to a tenant. The same user has different User IDs in different tenants. In one single tenant, a user has the same User ID in all apps （including store apps）. User ID is usually used to communicate user data between different apps. How to get User ID - `union_id`: Identifies a user to a tenant that acts as a developer. A user has the same Union ID in apps developed by the same developer, and has different Union IDs in apps developed by different developers. A developer can use Union ID to link the same user's identities in multiple apps. How to get Union ID - `email`: Identifies users by "email", which is the user's actual email address - `chat_id`: Identifies group chats by chat_id. You can call the API Search for groups visible to a user or bot to obtain the chat_id value. For group ID description, refer to Group ID description - `thread_id`: Identifies thread by thread_id.  For thread ID description,  refer to Thread Introduction **When the value is `user_id`, the following field scopes are required:** `contact:user.employee_id:readonly`  |
| `uuid` | `string` | No | A unique string sequence generated by the developer to forward message requests deduplicate; requests with the same uuid can only be successfully forwarded to the same target once within 1 hour. **Example value**: b13g2t38-1jd2-458b-8djf-dtbca5104204 **Data validation rules**: - Maximum length: `50` characters | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `receive_id` | `string` | Yes | According to the value of `receive_id_type`, fill in the ID of the corresponding forwarding target **Example value**: "ou_a0553eda9014c201e6969b478895c230" | ### Request body example

{
    "receive_id": "ou_a0553eda9014c201e6969b478895c230"
}

**cURL Example**

```json 
curl --location --request POST 'https://open.larksuite.com/open-apis/im/v1/messages/om_dc13264520392913993dd051dba21dcf/forward?receive_id_type=open_id' \
--header 'Authorization: Bearer Bearer t-XXX' \
--header 'Content-Type: application/json; charset=utf-8' \
--data-raw '{
    "receive_id": "ou_a0553eda9014c201e6969b478895c230"
}'
```
