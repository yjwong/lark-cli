# Reply a message

Replies to a specific message, with text, rich text, cards, group cards, name cards, images, videos, audios, files, and emojis supported.

> Notes:
> - Bot ability must be enabled.
> - To reply to private chat messages, the bot must be available  for the user.
> - To reply to group messages, the bot must be in the group chat.
> - To avoid disturbing users, the frequency limit for sending messages to the same user is 5 QPS, and the frequency limit for sending messages to the same group is 5 QPS shared by bots in the group.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/im/v1/messages/:message_id/reply |
| HTTP Method | POST |
| Rate Limit | 1000 per minute、50 per second |
| Supported app types | custom,isv |
| Required scopes Enable any scope from the list | `im:message` `im:message:send_as_bot` `im:message:send` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" How to choose and get access token |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Path parameters
| Parameter | Type | Description |
| --- | --- | --- |
| `message_id` | `string` | ID of the message to be replied to; for the description, refer to Message ID description **Example value**: "om_dc13264520392913993dd051dba21dcf" | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `content` | `string` | Yes | JSON format of message content; for format description, refer to Sent message content **Example value**: "`{\"text\":\"test content\"}`" |
| `msg_type` | `string` | Yes | Message type, which can be text, post, image, file, audio, media, sticker, interactive, share_card, or share_user **Example value**: "text" |
| `reply_in_thread` | `boolean` | No | Whether to reply in thread form; if the message you want to reply to is already a thread message, the reply will be in thread form by default. **Example value**: false **Default value**: `false` |
| `uuid` | `string` | No | A unique string sequence generated by the developer for replying to message requests deduplicate; requests with the same uuid are successfully executed at most once within 1 hour **Example value**: "a0d69e20-1dd1-458b-k525-dfeca4015204" **Data validation rules**: - Maximum length: `50` characters | ### Request body example

{
    "content": "{\"text\":\"test content\"}",
    "msg_type": "text",
    "reply_in_thread": true,
    "uuid": "a0d69e20-1dd1-458b-k525-dfeca4015204"
}

**Request Example**

```json 
curl --location --request POST 'https://open.larksuite.com/open-apis/im/v1/messages/om_xxxxxx/reply' \
--header 'Authorization: Bearer t-xxxxxx' \
--header 'Content-Type: application/json; charset=utf-8' \
--data-raw '{
    "content": "{\"text\":\"test content\"}",
    "msg_type": "text",
    "uuid": "a0d69e20-1dd1-458b-k525-dfeca4015204"
}'
```
