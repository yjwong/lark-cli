# Merge and forward messages

merges and forwards multiple messages from the same group to the specified user, group or thread.

> Notes:
> - Need to enable bot ability.
> - Merge and forward messages to the user, the bot needs to have availability for the user.
> - Merge and forward messages to the group, requiring the bot to be in the group.
> -  This API has the following restrictions:
> - Messages of type system are not supported for merging and forwarding.
> - Merging and forwarding messages from **different group chats** is not supported.
> - Does not support merging and forwarding messages from multiple threads at the same time.
> - It is not supported to merge and forward ordinary messages and messages in thread at the same time.
> - Merge and forward the sub-message (messages with  upper_message_id field) in the merge and forward message again is not supported.
> - The message content of the new message generated by merging and forwarding is a fixed value `Merged and Forwarded Message`, and its sub-messages can be obtained using the Obtain the content of the specified message API.
> - To avoid disturbing users, the frequency limit for sending messages to the same user is 5 QPS, and the frequency limit for sending messages to the same group is 5 QPS shared by bots in the group.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/im/v1/messages/merge_forward |
| HTTP Method | POST |
| Rate Limit | 1000 per minute、50 per second |
| Supported app types | custom,isv |
| Required scopes Enable any scope from the list | `im:message` `im:message:send_as_bot` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" How to choose and get access token |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Query parameters
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `receive_id_type` | `string` | Yes | Message receiver id type open_id/user_id/union_id/email/chat_id **Example value**: open_id **Optional values are**:  - `open_id`: Identifies a user to an app. The same user has different Open IDs in different apps. How to get Open ID - `user_id`: Identifies a user to a tenant. The same user has different User IDs in different tenants. In one single tenant, a user has the same User ID in all apps （including store apps）. User ID is usually used to communicate user data between different apps. How to get User ID - `union_id`: Identifies a user to a tenant that acts as a developer. A user has the same Union ID in apps developed by the same developer, and has different Union IDs in apps developed by different developers. A developer can use Union ID to link the same user's identities in multiple apps. How to get Union ID - `email`: Identifies users by "email", which is the user's actual email address - `chat_id`: Identifies group chats by chat_id. You can call the API Search for groups visible to a user or bot to obtain the chat_id value. For group ID description, refer to Group ID description - `thread_id`: Identifies thread by thread_id.  For thread ID description,  refer to Thread Introduction **When the value is `user_id`, the following field scopes are required:** `contact:user.employee_id:readonly`  |
| `uuid` | `string` | No | A unique string sequence generated by the developer for merge and forward message requests to deduplicate; requests with the same uuid can only be successfully forwarded once to the merge and forward to the same target within 1 hour. **Example value**: b13g2t38-1jd2-458b-8djf-dtbca5104204 **Data validation rules**: - Maximum length: `50` characters | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `receive_id` | `string` | Yes | According to the value of `receive_id_type`, fill in the ID of the corresponding merge and forward target **Example value**: "oc_a0553eda9014c201e6969b478895c230" |
| `message_id_list` | `string[]` | Yes | List of message IDs to forward.  For the description, refer to Message ID description. **Example value**: ["om_dc13264520392913993dd051dba21dcf"] **Data validation rules**: - Length range: `1` ～ `100` | ### Request body example

{
    "receive_id": "oc_a0553eda9014c201e6969b478895c230",
    "message_id_list": ["om_d862083dd13b61cc009e66c40a0f0a56","om_61244dab1ef11257621dcd7g0e256651"]
}

**cURL Example**

```json 
curl --location --request POST 'https://open.larksuite.com/open-apis/im/v1/messages/merge_forward?receive_id_type=chat_id' \
--header 'Authorization: Bearer t-XXX' \
--header 'Content-Type: application/json; charset=utf-8' \
--data-raw '{
    "message_id_list": ["om_d862083dd13b61cc009e66c40a0f0a56", "om_61244dab1ef11257621dcd7g0e256651"],
    "receive_id": "oc_a0553eda9014c201e6969b478895c230"
}

```
