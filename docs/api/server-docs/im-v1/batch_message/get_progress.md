# Queries the overall progress of a batch message

Based on the Query the number of users to whom a batch message is pushed and the number of users who have read the batch message
, this API increases the number of valid user ids in batch requests and the recall progress data of batch messages.

> Notes:
> - Only the messages generated by calling the Send a batch message API **within 30 days** can be queried.
> - The data returned by this API is the data captured when the query starts.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/im/v1/batch_messages/:batch_message_id/get_progress |
| HTTP Method | GET |
| Rate Limit | 1000 per minute、50 per second |
| Supported app types | custom,isv |
| Required scopes Enable any scope from the list | `im:message:send_multi_depts` `im:message:send_multi_users` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" How to choose and get access token | ### Path parameters
| Parameter | Type | Description |
| --- | --- | --- |
| `batch_message_id` | `string` | Task ID of the batch message to be queried, obtained by the return value of the Send messages in batches API. **Example value**: "bm-0b3d5d1b2df7c6d5dbd1abe2c91e2217" | ## Response

### Response body
| Parameter | Type | Description |
| --- | --- | --- |
| `code` | `int` | Error codes, fail if not zero |
| `msg` | `string` | Error descriptions |
| `data` | `\-` | \- |
|   `batch_message_send_progress` | `batch_message_send_progress` | Sending progress of the batch message |
|     `valid_user_ids_count` | `int` | Number of valid user IDs in the batch requests, including the user IDs of users who are invisible to bots. **Notes**:  The value of the valid_user_ids_count is 0 in the following cases: * The batch task has not been scheduled. Please wait a while and then call this API again. * When the batch message is sent, none of the passed openIDs, employeID, and departmentiIDs parameters include valid users. |
|     `success_user_ids_count` | `int` | Number of messages that are sent to users. **Note**: The value of the success_user_ids_count is not necessarily equal to the value of the valid_user_ids_count, because the valid_user_ids_count includes users who are invisible to bots. |
|     `read_user_ids_count` | `int` | Number of users who have read the batch message |
|   `batch_message_recall_progress` | `batch_message_recall_progress` | Recall progress of the batch message |
|     `recall` | `boolean` | Specifies whether the batch message is recalled. |
|     `recall_count` | `int` | Number of messages that are recalled | ### Response body example

{
    "code": 0,
    "msg": "success",
    "data": {
        "batch_message_send_progress": {
            "valid_user_ids_count": 204,

            "read_user_ids_count": 150
        },
        "batch_message_recall_progress": {

        }
    }
}
