# app_ticket event
> To understand the scenarios and configuration process of event subscription, see Event subscription overview.

For store apps, the open platform will push the app_ticket every other hour, which is used by the app to obtain the app_access_token.  

- Note: Store apps automatically subscribe to this event. Custom apps do not require this event.
- Used with: Trigger app_ticket re-push
 and Obtain token through app_ticket
]

**Callback example:**
```json
{
    "ts": "1502199207.7171419", //  The time when the event was sent, approximately the time when the event occurred. 
    "uuid": "bc447199585340d1f3728d26b1c0297a",  // Unique identifier of the event
    "token": "41a9425ea7df4536a7623e38fa321bae", // Verification token 
    "type": "event_callback", // This event is always event_callback.
    "event": {
         "app_id": "cli_xxx",
         "app_ticket":"xxx",
         "type":"app_ticket"
    }
}
```

	)
