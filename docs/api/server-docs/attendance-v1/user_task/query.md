# Obtain attendance results

Obtains the actual attendance results of employees in the Company, including the clock-in/out result.

> - If the shift set by the Company for an employee is clock-in at 9:00 a.m. and clock-out at 6:00 p.m., even if the employee submits attendance for several times during this period, the API will only return one record.
> - If you want to obtain detailed data of attendance, such as information of attendance location, you can use the API of "Obtain Attendance Flow Record" or "Batch Query of Attendance Flow Record".

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/attendance/v1/user_tasks/query |
| HTTP Method | POST |
| Supported app types | custom |
| Required scopes | `attendance:task:readonly` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" Learn more about obtaining and using access_token. |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Query parameters
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `employee_type` | `string` | Yes | Employee ID type **Example value**: "employee_id" **Optional values are**: - `employee_id`: Employee's employee ID, that is the user ID in Lark Admin > Organization > Member and Department > Member Details - `employee_no`: Employee ID, that is the employee ID in Lark Admin > Organization > Member and Department > Member Details |
| `ignore_invalid_users` | `boolean` | No | Whether to ignore invalid and unauthorized users. If true, the information of valid users is returned, with notification of information about invalid and unauthorized users. If false, and there are invalid or unauthorized users in user_ids, an error is returned **Example value**: true |
| `include_terminated_user` | `boolean` | No | Since new employees can reuse the employee_no/employee_id of resigned employees, if true, the data of all employees in service and resigned employees corresponding to employee_no/employee_id is returned. If false, only the data of the employees who are in service or the last resigned employee corresponding to employee_no/employee_id is returned **Example value**: true | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `user_ids` | `string[]` | Yes | employee_no or employee_id list **Example value**: abd754f7 |
| `check_date_from` | `int` | Yes | The start workday of the query **Example value**: 20190817 |
| `check_date_to` | `int` | Yes | The end workday of the query **Example value**: 20190820 | ### Request body example

```json
{
    "user_ids": [
        "abd754f7"
    ],
    "check_date_from": 20190817,
    "check_date_to": 20190820
}
```
