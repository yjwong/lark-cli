# Import attendance flow record

Imports the attendance flow record of authorized employees. After the import, the final attendance status and result can be calculated according to the shift rules of the employee's attendance group.

> Applicable to the attendance machine data import and other scenarios.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/attendance/v1/user_flows/batch_create |
| HTTP Method | POST |
| Supported app types | custom |
| Required scopes | `attendance:task` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" Learn more about obtaining and using access_token. |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Query parameters
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `employee_type` | `string` | Yes | Type of employee ID for user_id and creator_id in request body and response body **Example value**: "employee_id" **Optional values are**: - `employee_id`: Employee's employee ID, that is the user ID in Lark Admin > Organization > Member and Department > Member Details - `employee_no`: Employee ID, that is the employee ID in Lark Admin > Organization > Member and Department > Member Details | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `flow_records` | `user_flow[]` | Yes | Attendance flow record list |
| ∟ `user_id` | `string` | Yes | User ID **Example value**: "abd754f7" |
| ∟ `creator_id` | `string` | Yes | Record creator ID **Example value**: "abd754f7" |
| ∟ `location_name` | `string` | Yes | Attendance location name information **Example value**: "Xixi Ba Fang City" |
| ∟ `check_time` | `string` | Yes | Attendance time, with the timestamp accurate to seconds **Example value**: "1611476284" |
| ∟ `comment` | `string` | Yes | Attendance note **Example value**: "Clock-in" |
| ∟ `record_id` | `string` | No | Attendance record ID **Example value**: "6709359313699356941" |
| ∟ `longitude` | `float` | No | Attendance longitude **Example value**: 30.28991 |
| ∟ `latitude` | `float` | No | Attendance latitude **Example value**: 120.04513 |
| ∟ `ssid` | `string` | No | SSID of attendance Wi-Fi **Example value**: "b0:b8:67:5c:1d:72" |
| ∟ `bssid` | `string` | No | MAC address of attendance Wi-Fi **Example value**: "b0:b8:67:5c:1d:72" |
| ∟ `is_field` | `boolean` | No | Whether it is offsite attendance **Example value**: true |
| ∟ `is_wifi` | `boolean` | No | Whether Wi-Fi is used for attendance **Example value**: true |
| ∟ `type` | `int` | No | Record generation method **Example value**: 0 **Optional values are**: - `0`: User attendance - `1`: Administrator modification - `2`: User correction - `3`: Automatically generated by the system - `4`: Clock-out is not required - `5`: Attendance machine - `6`: Express attendance - `7`: Imports to attendance open platform |
| ∟ `photo_urls` | `string[]` | No | Attendance image list **Example value**: https://time.clockin.biz/manage/download/6840389754748502021 |
| ∟ `device_id` | `string` | No | Attendance device ID **Example value**: "99e0609ee053448596502691a81428654d7ded64c7bd85acd982d26b3636c37d" | ### Request body example
