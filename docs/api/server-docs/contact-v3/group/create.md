# Create a user group

This API is used to create a user group. Note that the app's contact scope must be "All employees" so as to create a user group. Click to learn about the range of contacts data that an app can access.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/contact/v3/group |
| HTTP Method | POST |
| Supported app types | custom |
| Required scopes | `contact:group` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" Learn more about obtaining and using access_token. |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `group_id` | `string` | No | User group ID, which can be customized when the user group is created, or will be automatically generated by the system if not customized. The group_id of user groups cannot be modified once created. Data validation rules for a custom group_id: Maximum length: 64 characters Validation rules: A combination of numbers and uppercase and lowercase letters without spaces. **Example value**: "g122817" |
| `name` | `string` | Yes | User group name, which is unique in a company. The maximum length is 100 characters. **Example value**: "IT outsourcing group" |
| `description` | `string` | No | User group description **Example value**: "Collection of IT service members" |
| `type` | `int` | No | User group type. It defaults to 1, which indicates common user group. **Example value**: 1 **Optional values are**:  - `1`: Common user group  **Default value**: `1` | ### Request body example

{
    "group_id": "g122817",
    "name": "IT outsourcing group",
    "description": "Collection of IT service members",
    "type": 1
}

## Response

### Response body

| `code` | `int` | Error codes, fail if not zero |
| `msg` | `string` | Error descriptions |
| `data` | `\-` | \- |
|   `group_id` | `string` | User group ID | ### Response body example

{
    "code": 0,
    "msg": "success",
    "data": {
        "group_id": "g122817"
