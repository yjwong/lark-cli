#  Overview
##  Resource definition
A user of a company in Lark.

##  Field description

| Parameter | Type | Description |
| --- | --- | --- |
| `user_id` | `string` | Unique identifier of a user in a tenant, which can be customized or randomly generated by the system. For the detailed definition, refer to What is User ID?; for how to obtain it, refer to How do I obtain User ID, Open ID, and Union ID? **Data validation rules for a custom `user_id`**: - Maximum length: `64` characters - Regular validation: No spaces are included **Example value**: "u273y71" **Required field scopes**: Obtain user ID |
| `open_id` | `string` | Unique identifier of a user in a tenant. For the detailed definition, refer to What is Open ID?; for how to obtain it, refer to How do I obtain User ID, Open ID, and Union ID? **Example value**: "ou_7dab8a3d3cdcc9da365777c7ad535d62" |
| `union_id` | `string` | Unique identifier of a user in a tenant. For the detailed definition, refer to What is Union ID?; for how to obtain it, refer to How do I obtain User ID, Open ID, and Union ID? **Example value**: "on_cad4860e7af114fb4ff6c5d496d1dd76" |
| `name` | `string` | User name **Example value**: "John" **Data validation rules**: - Minimum length: `1` character **Required field scopes (Satisfy any)**: Obtain user's basic information Read Contacts as an app |
| `en_name` | `string` | English name **Example value**: "John" **Required field scopes (Satisfy any)**: Obtain user's basic information Read Contacts as an app |
| `email` | `string` | Email **Example value**: "zhangsan@gmail.com" **Required field scopes**: Obtain user's email information |
| `mobile` | `string` | Mobile number **Example value**: "13011111111" **Required field scopes**: Obtain user's mobile number |
| `mobile_visible` | `boolean` | Visibility of mobile number. true: visible (default), false: invisible. If it is set to false, the employee's mobile number will be invisible to other company members. **Example value**: false |
| `gender` | `int` | Gender **Example value**: 1 **Optional values are**: - `0`: Unknown - `1`: Male - `2`: Female **Required field scopes (Satisfy any)**: Obtain user's gender Read Contacts as an app |
| `avatar_key` | `string` | File key of the profile photo. You can upload an image by calling the "Upload Image" API in "Messenger > Message > Images" to obtain the file key. **Example value**: "2500c7a9-5fff-4d9a-a2de-3d59614ae28g" |
| `department_ids` | `string[]` | List of IDs of the user's departments **Example value**: od-4e6ac4d14bcd5071a37a39de902c7141 |
| `leader_user_id` | `string` | User ID of the user's direct manager **Example value**: "ou_7dab8a3d3cdcc9da365777c7ad535d62" |
| `city` | `string` | City **Example value**: "Hangzhou" **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| `country` | `string` | Country or region code. For the specific code, refer to the country/region code list. **Example value**: "CN" **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| `work_station` | `string` | Seat ID **Example value**: "North building-H34" **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| `join_time` | `int` | Date of employment **Example value**: 2147483647 **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| `employee_no` | `string` | Employee ID **Example value**: "1" **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| `employee_type` | `int` | Employee type. Optional values are: - 1: Regular - 2: Intern - 3: Outsourcing - 4: Contractor - 5: Consultant The int value of the custom employee type can also be read, and the name of the custom employee type for the tenant can be obtained through the API: Obtain workforce type **Example value**: 1 |
| `orders` | `user_order[]` | User sorting information |
| ∟ `department_id` | `string` | Department ID **Example value**: "od-4e6ac4d14bcd5071a37a39de902c7141" |
| ∟ `user_order` | `int` | The user's ranking in their direct department. A larger value indicates a higher ranking. **Example value**: 100 |
| ∟ `department_order` | `int` | Ranking of the user's departments. A larger value indicates a higher ranking. **Example value**: 100 |
| `custom_attrs` | `user_custom_attr[]` | Custom field **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| ∟ `type` | `string` | Custom field type - `TEXT` - `HREF` - `ENUMERATION` - `PICTURE_ENUM` - `GENERIC_USER` **Example value**: "TEXT" |
| ∟ `id` | `string` | Custom field ID **Example value**: "DemoId" |
| ∟ `value` | `user_custom_attr_value` | Custom field value. For details, refer to the FAQs. |
| ∟ `text` | `string` | When the field type is TEXT, this parameter defines the field value; when the field type is HREF, this parameter defines the webpage title. **Example value**: "DemoText" |
| ∟ `url` | `string` | When the field type is HREF, this parameter defines the default URL. **Example value**: "http://www.larksuite.com" |
| ∟ `pc_url` | `string` | When the field type is HREF, this parameter defines URL on the PC. **Example value**: "http://www.larksuite.com" |
| ∟ `option_id` | `string` | When the field type is ENUMERATION or PICTURE_ENUM, this parameter defines the option value. **Example value**: "edcvfrtg" |
| ∟ `generic_user` | `custom_attr_generic_user` | When the field type is GENERIC_USER, this parameter defines the user referenced. |
| ∟ `id` | `string` | User's user_id **Example value**: "9b2fabg5" |
| ∟ `type` | `int` | User type    1: User **Example value**: 1 |
| `enterprise_email` | `string` | Business email. Make sure that Lark Business Mail has been enabled in Admin. **Example value**: "demo@mail.com" **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| `job_title` | `string` | Job title **Example value**: "xxxxx" **Required field scopes (Satisfy any)**: Obtain user's employment information Read Contacts as an app |
| `need_send_notification` | `boolean` | Indicates whether to send a notification **Example value**: false |
| `notification_option` | `notification_option` | Specifies how to invite users |
| ∟ `channels` | `string[]` | Channel list. Enum values are: sms (Invite via text message), email (Invite via email) **Example value**: ["sms", "email"] |
| ∟ `language` | `string` | Language **Example value**: "zh-CN" **Optional values are**: - `zh-CN`: Chinese - `en-US`: English - `ja-JP`: Japanese | ### Data example
