# Update a user’s information
> In order to improve the security level, we have upgraded it. Please migrate to the new version as soon as possible. New version >>

Use this API to update a user’s information in Contacts. 

- The application should apply for `update contacts` and `access contacts by application identity` permissions to invoke this API. Permission to write to Contacts for this user and related departments is required. To invoke this API, the request header must specify the tenant_access_token. Marketplace apps do not have permission to invoke this API.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/contact/v1/user/update |
| HTTP Method | POST |
| Required scopes Enable any scope from the list | update contacts access contacts by application identity | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value Format**: "Bearer `access_token`" **Example Value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" Learn more about obtaining and using access_token. |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Request body   
|Parameter|Type|Mandatory|Description|
|--|-----|--|----|
|employee_id、open_id|String|Yes|User’s open ID or employee ID|
|name|String|Yes|User’s name|
|email|String|No|User's email address, can't update both email and mobile empty|
|mobile|String|No|User's mobile number. Can't update user mobile empty if user is unactivated, else can't update both email and mobile empty|
|department_ids|list|Yes|User's departments. The user can be added to multiple departments at the same time, but Contacts permission is required for all the departments|
|city|String|No|User's city|
|country|String|No|country|
|gender|String|No|Gender. 1: male, 2: female|
|employee_type|Int|No|Employee type. 1: Regular employee, 2: Intern, 3:Freelancer, 4: Agency worker, 5: Consultant|
|join_time|String|No|The date the employee joined the company|
|leader_employee_id、leader_open_id|String|No|Line manager open ID or employee ID|
|employee_no|String|No|User Work No.|
|need_send_notification|Bool|No|If an invitation is sent. If true, an invitation will be sent to the user’s mail address or mobile phone after the user is added|
|custom_attrs|object|No|Custom user properties.This field is only valid if the organization administrator has enabled "Allow the Open Platform API to be invoked". Each custom user property passed must include the property ID generated by the platform and the value you want to set.If "Allow the Open Platform API to be invoked" is not enabled, or the custom property ID does not exist or is illegal, this information will be ignored.|
|work_station|String|No|Seat|
