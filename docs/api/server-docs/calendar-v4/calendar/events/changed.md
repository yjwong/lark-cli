# Calendar changes

After the user subscribes to the calendar change event, if there is a calendar change in the user's calendar list, this event will be triggered.{Usage Examples}(url=/api/tools/api_explore/api_explore_config?project=calendar&version=v4&resource=calendar&event=changed)

> - You need to subscribe to the calendar change event for the user first, and configure the event subscription in the application, so that you can receive event data when the event is triggered. To learn about event subscriptions, refer to the Overview of Event Subscriptions.
> - This event mainly contains user information about the calendar change, not calendar information. Therefore, when you receive the event request, you should extract the user information from the user_id_list field, and then use these user identities (user_access_token) to call the Query Calendar List interface, to query the incremental change data of the calendar.
