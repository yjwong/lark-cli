# Event resource introduction

A event is an instance resource that exists in the calendar. You can build a work event for specified topics by associating rules such as specific dates or time periods, invitees, locations, etc. For example, personal work reminders, team meeting communications, event live broadcasts, etc. You can build and manage events through the event resource API. This article will introduce the property information used to define the detailed configuration of the event.

## Event organizer
Each event has an organizer, which represents the calendar in which the event was created. For example:

- The application uses the `tenant_access_token` to create an event on the main calendar, and the organizer of the event is the application's main calendar.
- The application uses a user's `user_access_token` to create an event on the primary calendar, and the organizer of the event is the user's primary calendar.
- If the event is created on a shared calendar, the shared calendar is the event organizer.

Event organizers and users with permissions on the organizer's calendar have full permissions on the event.

## Repeat event
Repeating events will occur multiple times according to the specified recurrence rules, resulting in multiple instances, such as weekly meetings, daily meetings, etc. These event instances usually have the same title, description, invitees, etc.

### Repeatability rules
The time of a recurring event is defined by two pieces of data:
- The `start` and `end` fields of the event define the time when the repeating event is repeated each time.
- The `recurrence` field of the event defines the rules for recurring events.

The `recurrence` field of the event is defined using `RRULE` in [RFC5545](https://datatracker.ietf.org/doc/html/rfc5545#section-3.8.5.3). `RRULE` generally consists of the following parts :
- `FREQ`: Frequency of event repetition, required. For example `DAILY` or `WEEKLY`.
- `INTERVAL`: cooperates with `FREQ` to refine frequency information. For example `FREQ=DAILY;INTERVAL=2` means repeat every two days.
- `COUNT`: The number of times the event is repeated.
- `UNTIL`: The last time the event will repeat.
- `BYDAY`: a specific date on which the event repeats. For example, `FREQ=WEEKLY;BYDAY=MO` means it repeats every Monday.
> **Note:** `COUNT` and `UNTIL` cannot appear together in `RRULE`.

### Exception event

A recurring event is composed of multiple event instances at different times. When the information of an instance is modified and is different from the recurring event, the instance is called an exception event. Exception events may have different titles, descriptions, start times, invitees and other information from the original duplicate events, and exception events can also be deleted individually.

When the `status` of an exception event is `cancelled`, it indicates that this instance of the repeated event has been canceled and should no longer be displayed to the user. Applications should bind exception events with `status` as `cancelled` to the associated duplicate events to avoid erroneous display information.
