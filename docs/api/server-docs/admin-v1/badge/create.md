# Create a badge

Use this API to create a complete badge information, up to 1000 badges can be created under a tenant.

## Request
| Facts |  |
| --- | --- |
| HTTP URL | https://open.larksuite.com/open-apis/admin/v1/badges |
| HTTP Method | POST |
| Rate Limit | 100 per minute |
| Supported app types | custom |
| Required scopes | `admin:badge` | ### Request header
| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| Authorization | string | Yes | `tenant_access_token` **Value format**: "Bearer `access_token`" **Example value**: "Bearer t-7f1bcd13fc57d46bac21793a18e560" How to choose and get access token |
| Content-Type | string | Yes | **Fixed value**: "application/json; charset=utf-8" | ### Request body

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | `string` | Yes | Unique badge name within the tenant, up to 30 characters. **Example value**: "Incentive badge." |
| `explanation` | `string` | No | Description copy of the badge, up to 100 characters. **Example value**: "This badge is awarded to motivate employees." |
| `detail_image` | `string` | Yes | The details of the enterprise badge Key. 1. Permission verification: The picture key uploaded by the non-tenant cannot be used directly; 2. Time limit verification: When creating a badge or modifying the image key of the badge, you need to use the image key uploaded within 1h. **Example value**: "75a1949f-d9df-4b46-bc88-dacc51e88f3j" **Data validation rules**: - Minimum length: `1` characters |
| `show_image` | `string` | Yes | Key of the profile decoration image of the enterprise badge. 1. Permission verification: The picture key uploaded by the non-tenant cannot be used directly; 2. Time limit verification: When creating a badge or modifying the image key of the badge, you need to use the image key uploaded within 1h. **Example value**: "03daa74a-159f-49e9-963e-b6c4d76103fj" **Data validation rules**: - Minimum length: `1` characters |
| `i18n_name` | `i18n` | No | The multi-language name of the badge, the same as the name field limit, with a maximum of 30 characters. **Example value**: {         "zh_cn": "激励勋章",         "en_us": "Incentive Medal",         "ja_jp": "奨励メダル"     } |
|   `zh_cn` | `string` | No | Chinese. **Example value**: "激励勋章" **Data validation rules**: - Minimum length: `1` characters |
|   `en_us` | `string` | No | English. **Example value**: "Incentive Medal" **Data validation rules**: - Minimum length: `1` characters |
|   `ja_jp` | `string` | No | Japanese. **Example value**: "奨励メダル" **Data validation rules**: - Minimum length: `1` characters |
| `i18n_explanation` | `i18n` | No | The multi-language description copy of the badge, the same as the explanation field limit, has a maximum of 100 characters. **Example value**: {         "zh_cn": "这枚勋章为了激励员工颁发。",         "en_us": "This medal is awarded to motivate employees.",         "ja_jp": "このメダルは、従業員のモチベーションを高めるために授与されます。"     } |
|   `zh_cn` | `string` | No | Chinese. **Example value**: "激励勋章" **Data validation rules**: - Minimum length: `1` characters |
|   `en_us` | `string` | No | English. **Example value**: "Incentive Medal" **Data validation rules**: - Minimum length: `1` characters |
|   `ja_jp` | `string` | No | Japanese. **Example value**: "奨励メダル" **Data validation rules**: - Minimum length: `1` characters | ### Request body example

{
    "name": "Incentive badge.",
    "explanation": "This badge is awarded to motivate employees.",
    "detail_image": "75a1949f-d9df-4b46-bc88-dacc51e88f3j",
    "show_image": "03daa74a-159f-49e9-963e-b6c4d76103fj",
    "i18n_name": {
        "zh_cn": "激励勋章",
        "en_us": "Incentive Medal",
        "ja_jp": "奨励メダル"
    },
    "i18n_explanation": {
        "zh_cn": "激励勋章",
        "en_us": "Incentive Medal",
        "ja_jp": "奨励メダル"
    }
}
